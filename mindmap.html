—— 分流结果 ——
分流逻辑：每道题的每个选项有对应的分值权重（ad / agency / content），答完后累加总分取最高。打平时按答题顺序取优先信号。
第 1 题的特殊作用：不只参与分流计分，还决定结果页的"核心对比句"——拿用户现在的方式和推荐方式做直接对比。

直接起量型

→ 广告投放方案

一句话：先快速验证，有线索再说。

典型用户：打电话找客户 + 想尽快有线索 + 没内容能力 + 想自己把控

对比句示例："相比一个一个打电话，广告投放能让客户主动找上门。"

专业代跑型

→ 专业代投方案

一句话：自己扛执行容易断，交给团队更稳。

典型用户：转介绍为主 + 想省心 + 内容没章法 + 想交给团队

对比句示例："转介绍量不稳定，让专业团队做投放可以持续补充新客源。"

经营放大型

→ 内容经营 + 投流方案

一句话：先搭好基础，再配合投流放大。

典型用户：已在发内容 + 想打基础 + 有平台经验 + 想协作推进

对比句示例："您已经有内容基础，下一步是把内容、承接和投放串成完整链路。"

—— 结果页结构 ——
S1 类型标题（直接起量型 / 专业代跑型 / 经营放大型）+ 匹配度分数
↓ 核心对比句（基于第 1 题选的获客方式 × 结果类型，直接对比现有方式和推荐方式）
S2 结合您的生意情况（卡片式：当前获客方式 / 经营形式 / 门店规模 / 内容能力 / 推进偏好 / 对应方案）
S3 分析（2-3 条判断依据，按内容能力和经营形式动态组合）
S4 怎么推进（3 条行动建议，第 1 条按经营形式变化）
→ 加顾问区："看看同行怎么做的：花了多少、拿了多少线索、用了哪种方案"
—— 转化闭环 ——
信息流广告 → H5 首页 → 5-6 题答题 → 结果页（对比 + 分析 + 建议）→ 扫码加顾问 → 领取同行案例和详细方案
